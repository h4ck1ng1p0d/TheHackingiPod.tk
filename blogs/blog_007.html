KeePass Local Password Manager

Aug 20th, 2021

How I found out about keepass

recently, I’ve had some corrupted sections in my boot partition and I was forced to reinstall Ubuntu again. Ugh, this took a little bit but luckily I recovered all my data.

But after that I started looking at some good backup solution so if this happens again to me in the future I won’t have to worry as much about recovering everything. But I got sidetracked and came on backing up my credentials for all my accounts.

At the moment I’m using Bitwarned, a friend of mine recommended for me because and this are his words: because of the geeky features. And after he told me that I was sold and moved all my account credentials to Bitwarned and it is working out really good, but I started researching for redundant options so if I did lose my access to Bitwarned I’ve got a backup somewhere else.

And after researching for a couple of hours I’ve found KeePass V2, This is an old password manager from I think the beginning of the 2000s but it’s pretty cool, Let me explain.

When you install keepass you will be prompted to create a database and configuring how you want to access it, it look intimidating but if you look good enough you can configure some pretty cool and geeky settings.

For example you can choose to use a standard password like most password managers or 1 of my favorite features is generating a key file which you can store on a USB flash drive and just plug in and keepass with remember where you stored it.


Encryption and Login Settings

the encryption is standard AES-256 encryption for every database. But you for the key file you can make it as strong as you want so you can also generate really weak key and use that, but I wouldn’t recommend it.

These are the login options to access your database:

  1. Normal Password –> Just a simple passphrase to unlock you database
  2. Key File –> A key file which you can store on a external storage device and just plug it in to unlock your database or vault.
  3. Windows User Password --> This is a really good option for windows users because you can use the same password from their local windows account which you use to unlock you PC you can also for your vault.
I do not recommend this option because windows local accounts password are really easy to bypass and a hacker could change the password and access your PC and your vault.


What I use it for.

After creating a database you will be presented with a sample page showing you how to create, delete or edit keys. But one thing I love about keepass is their network section, this section I assume is used to store server credentials from Linux servers and such. But I use it to store Network credentials that I’ve collected from people and from vulnerable WiFi networks.


What you can use it for.

I recommend using keepass for it’s local storing of the database on your computer or USB flash drive avoiding storing your credentials on some server in a data center. The key file unlocking, not many password manager allow you to use a physical 2FA key such as the Yubikey or just a basic flash drive and even if they do you might have to upgrade to a plan which you might have to pay for.
